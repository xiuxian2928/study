Named Function (기명함수)

Named Function은 말그대로 사용자가 이름을 지정한 함수로, 선언적 함수라고도 부르는 함수이다.

function nameFunction(){}

nameFunction();

Anonymouse Function 과 Named Function의 차이

두 함수는 브라우저가 읽는 순서의 차이가 있다.

Anonymouse Function 같은 경우는 브라우저가 런타임(RunTime)에 동적으로 선언되는 함수입니다.

Named Function 같은 경우는 브라우저가 런타임(RunTime) 이전에 선언되는 함수입니다.

ex)

// 기명함수가 앞으로, 무명함수가 뒤로 function func(){ console.log('기명 함수 입니다.''); }
var func = function(){ console.log('무명 함수 입니다.''); }
func();

// 무명함수가 앞으로, 기명함수가 뒤로 var func2 = function(){ console.log('무명 함수 입니다.''); }
function func2(){ console.log('기명 함수 입니다.''); }
func2();

둘다 같은 이름으로 선언하고 호출 했을 때 실행되는 함수는 둘다 무명함수이다.
이렇듯, Anonymouse Function이 Named Function 이후에 실행이 되므로 익명함수가 실행되게 되는 것이다.

Arrow Function

Arrow Function은 익명 함수를 좀 더 간결하게 표현할 수 있도록 ES2015에서 새롭게 등장한 함수 선언 방식이다.

ex)

// arrow function 정의 const getTwice = (number) => { return number * 2; };
// 콜백 함수로 활용 myBtn.addEventListener('click', () => { console.log('button is clicked!); };

Arrow Function은 다양한 상황에 따라 축약형으로 작성될 수 있다.

// 1. 함수의 파라미터가 하나 뿐일 때 const getTwice = (number) => { return number * 2; };
// 파라미터를 감싸는 소괄호 생략 가능 const getTwice = number => { return number * 2; };
// 2. 함수 동작 부분이 return문만 있을 때 const sum = (a, b) => { return a + b; };
// return문과 중괄호 생략 가능 const sum = (a, b) => a + b;

그리고 Arrow function이 일반 함수와 몇가지 차이점이 있다.

가장 대표적인 차이점은 arguments 객체가 없고, this가 가리키는 값이 일반 함수와는 다른 점이다.

arguments 객체

자바스크립트 함수 안에는 arguments라는 독특한 객체가 존재한다.

arguments 객체는 함수를 호출할 때 전달한 아규먼트들을 배열의 형태로 모아둔 유사 배열 객체이다. 특히, 함수를 호출할 때 전달되는 아규먼트의 개수가 불규칙적일 때 유용하게 활용될 수 있다.

function printArguments(){ // arguments 객체의 요소들을 하나씩 출력 for (const arg of arguments){ console.log(arg) } }
printArguments('Young', 'Mark', 'Koby'); // 'Young', 'Mark', 'Koby'가 콘솔에 하나씩 출력됨

this

자바스크립트에는 this라는 조금 특별한 키워드가 있다.

웹 브라우저에서 this가 사용될 때는 전역 객체, Window 객체를 가지게 된다. 하지만 객체의 메소드를 정의하기 위한 함수 안에선 메소드를 호출한 객체를 가리키게 된다.

const = { firestName: 'Tess' lastName: 'Jang', getFullName: function(){ return `${this.firestName} ${this.lastName}`; }, }
